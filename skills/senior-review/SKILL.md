---
name: senior-review
description: Review a code changes, PR, etc. with a particularly focus on long-term software maintenance, architecture and code duplication. Use when user wants to carefully review a code change, or has a large PR they're about to submit to others for review.
---

Review the specified code change/PR, ideally by checking it out locally as a worktree (re-use the existing branch if already available locally). What would a senior engineer who is particularly concerned with code duplication, long-term maintenance and software architecture, change about this implementation? Validate technical assumptions with sub-agents.

Don't take the design (including requirements, e.g. a Linear Issue) as fixed, it can be modified to facilitate improvements and reduce software coupling or complexity.

If the specific code to review is unclear, ask.
